1. Brand overview
Brand reference: Google Play Store – Apps section listing page (web).
Usage intent: Use this document as a prompt-compatible design system to generate HTML/CSS/JS frontends (or any web UI framework) that visually align with the Google Play Store brand and layout patterns.

Core visual concepts:

Clean, white canvas: Content-first, lots of whitespace, subtle separators.

Google Play accent: Play Green plus Google core colors for small accents.

Rounded cards: Apps as tiles/cards with icon, name, rating, category; soft shadows.

Grid/list hybrid: Responsive grid of cards on desktop, single-column or 2-column on mobile.

Subtle motion: Hover elevation, ripple feedback, smooth carousels.

All colors are defined as CSS custom properties so an AI can drop them into any stack.

Usage rules:

Background: --color-bg for main page; --color-bg-alt for toolbars/filters.

Primary CTA buttons: --color-primary background, --color-text-on-primary text.

Text hierarchy: Titles use --color-text-primary, subtitles --color-text-secondary, metadata --color-text-muted.

Dividers: 1px lines using --color-divider or --color-border-subtle.

This palette aligns with the published Google Play and Google brand colors.

4. Layout system and page structure
This section describes the structural layout of the Google Play Apps listing-style page so it can be re-created.

4.1 Global page layout
Key patterns:

Sticky top bar: Logo, search, nav, account.

Main content width: Centered, max-width ~1200–1320px.

Carousel JS behavior (structural pattern)
js
document.querySelectorAll('.gp-carousel').forEach(carousel => {
  const track = carousel.querySelector('.gp-carousel-track');
  const prev = carousel.querySelector('.gp-carousel-arrow-left');
  const next = carousel.querySelector('.gp-carousel-arrow-right');

  const scrollAmount = 300;

  if (prev && next && track) {
    prev.addEventListener('click', () => {
      track.scrollBy({ left: -scrollAmount, behavior: 'smooth' });
    });

    next.addEventListener('click', () => {
      track.scrollBy({ left: scrollAmount, behavior: 'smooth' });
    });
  }
});
5.5 Hover, focus, and “ripple” interactions
Play uses Material-like ripple on click; structurally you can approximate with JS + CSS.

Generic ripple container
css
.gp-ripple {
  position: relative;
  overflow: hidden;
}

.gp-ripple::after {
  content: "";
  position: absolute;
  border-radius: 50%;
  transform: scale(0);
  opacity: 0;
  pointer-events: none;
}

.gp-ripple-active::after {
  animation: gp-ripple 400ms ease-out;
}

@keyframes gp-ripple {
  from {
    transform: scale(0);
    opacity: 0.3;
    background: var(--color-ripple);
  }
  to {
    transform: scale(8);
    opacity: 0;
    background: transparent;
  }
}
JS to trigger ripple on buttons/cards
js
document.addEventListener('click', e => {
  const target = e.target.closest('.gp-ripple');
  if (!target) return;

  target.classList.remove('gp-ripple-active');
  void target.offsetWidth; // reflow to restart animation
  target.classList.add('gp-ripple-active');
});
Apply .gp-ripple to buttons/cards:

html
<button class="gp-btn gp-btn-primary gp-ripple">Install</button>
<article class="gp-app-card gp-ripple">...</article>
